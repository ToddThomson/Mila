# Data Directory

This directory contains all datasets, models, scripts, and tools for the Mila project.

## Structure

### `/Datasets`
Contains training and evaluation datasets. Each dataset follows a standardized structure:
- `raw/` - Original, unprocessed data files
- `encoded/` - Tokenized output generated by the Mila tokenize tool
- `vocabularies/` - Tokenizer vocabulary files (`.bpe.mila`, `.char.mila`)

**Current Datasets:**
- `Mnist/` - MNIST handwritten digits dataset
- `Shakespeare/` - Shakespeare text corpus

### `/Models`
Pretrained open-source models for evaluation and fine-tuning. These are produced by Python conversion scripts and represent checkpoint snapshots ready for use in C++.

**Current Models:**
- `Gpt2/` - GPT-2 model weights and configuration

**Planned:**
- LLaMA variants
- Mistral variants

### `/Scripts`
Python utilities for converting between Python and C++ formats. Handles tokenizer vocabularies and model weight conversions to Mila-compatible formats.

### `/Tools`
Executables and utilities for data processing.
- Contains the Mila tokenize tool for dataset preprocessing

## Workflow

1. **Dataset Preparation**: Place raw data in `Datasets/<name>/raw/`
2. **Vocabulary Creation**: Generate tokenizer vocabularies (BPE/char) and store in `Datasets/<name>/vocabularies/`
3. **Tokenization**: Use Mila tokenize tool to create encoded datasets in `Datasets/<name>/encoded/`
4. **Model Conversion**: Run Python scripts to convert pretrained models to Mila format in `/Models`

## Notes

- All model assets are generated from Python conversion scripts
- Vocabulary files use `.mila` extension with tokenizer type prefix (e.g., `dataset.bpe.mila`)
- Future model organization may include versioning structure for checkpoints and variants
